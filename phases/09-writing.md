### Step 9.4：正文撰写（占位符策略）
### Step 9.4：叙事结构指导

撰写前，AI 应向用户说明每一章的**叙事目标**，确保全文逻辑连贯：

```
各章节叙事结构
═══════════════════════════════════════

第一章 引言 —— "为什么重要 + 为什么可信"
  叙事目标：让读者在 2 页内理解研究的动机、方法和贡献
  关键问题：
  • 现实问题是什么？现有研究有什么不足？
  • 本文用什么方法解决？为什么这个方法可信？
  • 本文的核心发现是什么？有什么贡献？
  常用结构：漏斗形（宽 → 窄 → 宽）
  背景 → 文献缺口 → 本文方案 → 发现 → 贡献

第二章 文献综述 —— "定位本研究"
  叙事目标：画出学术地图，让读者知道本文的位置
  关键问题：
  • 前人做了什么？主要结论是什么？
  • 还有什么没做或做得不够好？
  • 本文的边际贡献在哪里？
  注意：不是文献堆砌，而是按主题/逻辑组织

第三章 理论机制 —— "为什么 X 会影响 Y"
  叙事目标：让读者相信 X→Y 有理论基础
  关键问题：
  • 理论上 X 为什么会影响 Y？
  • 通过什么渠道？条件是什么？
  • 是否有正式的理论模型？

第四章 研究设计 —— "让读者相信识别策略是有效的"
  叙事目标：说服审稿人 X 的外生变异是可信的
  关键问题：
  • 数据来源是否可靠？样本选择是否有偏？
  • 变量的测量方式是否合理？
  • 识别策略为什么能识别因果关系？
  • 识别假设是否可检验？如何检验？

第五章 实证结果 —— "呈现证据并排除替代解释"
  叙事目标：用证据说话，让读者无法拒绝结论
  关键问题：
  • 基准结果是否稳健？
  • 除了 X→Y，还有没有其他可能？
  • 机制是什么？异质性体现在哪里？

第六章 结论 —— "总结贡献 + 指出局限"
  叙事目标：让读者记住核心发现
  关键问题：
  • 核心发现是什么？经济含义是什么？
  • 政策含义是什么？
  • 有哪些局限性？外部有效性如何？
```

### Step 9.5：实证部分写作顺序

推荐按以下顺序撰写，从最确定的部分开始：

```
推荐写作顺序
═══════════════════════════════════════

第一阶段：写结果（最确定，避免"先写背景再改结果"）
  → 第五章 实证结果分析（基准 → 内生性 → 稳健性 → 机制 → 异质性）
  → 第四章 研究设计（数据 → 变量 → 模型设定）

第二阶段：写框架（理论支撑）
  → 第三章 理论机制与研究假设
  → 第二章 文献综述

第三阶段：写总结（统领全文，最后写）
  → 第一章 引言（最后写，因为需要知道全文内容才能提炼）
  → 第六章 结论与政策建议

第四阶段：写摘要（最浓缩，最后写）
  → 中文摘要 → 英文摘要

⚠️ 常见错误：先写引言再写结果 → 等结果出来后发现引言说的跟实际不符，
   需要反复修改。推荐"结果先行，引言收尾"。
```

### Step 9.6：正文撰写（占位符策略）

正文撰写采用 **`[CIT:关键词]` 占位符**策略：

- 所有引用位置写作：`研究指出，X 对 Y 有显著正向影响[CIT:Author2020_keyfinding]`
- 占位符格式：`[CIT:关键词1;关键词2]`（用于需要合并多篇文献的表述）
- 不插入具体引文格式，避免后续替换时污染格式
- 每章分步骤撰写，完成后用户可以分段确认

撰写顺序：
1. 实证结果章节（第五章）— 最确定，最先写
2. 研究设计章节（第四章）
3. 理论机制章节（第三章）— 结合 Phase 2
4. 文献综述章节（第二章）
5. 引言章节（第一章）— 最后写，统领全文
6. 结论章节（第六章）
7. 中英文摘要 — 最后浓缩


### Step 9.8：引文标注（占位符替换）

用户补充文献后，AI 重新精读所有引用到的 PDF，提取准确引用信息：

```
引用信息标注（每条占位符替换）
─────────────────────────────────
[CIT:Author2020_keyfinding]
  → (王某某等, 2020) 或
  → (Smith et al., 2020) 或
  → 多篇合并时：研究指出...（王某某等, 2020; Smith et al., 2020）
─────────────────────────────────
```

批量替换所有 `[CIT:...]` 占位符为 `(Author, Year)` 格式。同时生成参考文献列表（GB/T 7714 格式）。

图表排版格式（三线表标准）：
- 表题在上：小五号黑体，居中
- 表格为三线表（顶线 1.5pt / 栏目线 0.75pt / 底线 1.5pt）
- 表注在下：小五号宋体
- 图题在下：小五号黑体，居中

### 输出

论文最终输出为 `thesis/论文题目_终稿.docx`。

### A 路线（Python python-docx）示例

```python
from docx import Document
from docx.shared import Pt, Cm, Inches
from docx.enum.text import WD_ALIGN_PARAGRAPH

doc = Document()

# 页面设置
section = doc.sections[0]
section.page_width = Cm(21)
section.page_height = Cm(29.7)
section.left_margin = Cm(3)
section.right_margin = Cm(2.5)
section.top_margin = Cm(2.5)
section.bottom_margin = Cm(2.5)

# 一级标题
h1 = doc.add_heading("第一章 引言", level=1)
h1_run = h1.runs[0]
h1_run.font.name = '黑体'
h1_run.font.size = Pt(14)

# 正文段落
p = doc.add_paragraph()
p.style = doc.styles['Normal']
p_run = p.runs[0]
p_run.font.name = '宋体'
p_run.font.size = Pt(12)
p.paragraph_format.line_spacing = 1.5

# 插入图片
doc.add_picture("output/figures/dist.png", width=Cm(12))

# 保存
doc.save("thesis/论文题目_终稿.docx")
```
